# 银行客户流失分析项目

## 一、项目简介

本项目围绕银行客户流失问题，基于 10,000 条客户数据开展多维度分析。项目使用 MySQL 统计整体流失情况，并从地区、性别、年龄、资产、产品持有数量和客户活跃状态等维度识别高流失人群，最终通过可视化大屏和商业分析报告呈现分析结论与客户挽留建议。

项目形成了从数据明细、SQL 查询、指标结果、可视化展示到商业分析报告的完整分析闭环，可用于数据分析作品集展示。

## 二、项目目标

1. 统计客户总量、流失客户数量和整体流失率。
2. 对比不同客户群体的流失差异，定位高风险客群。
3. 将常用分析结果整理为独立数据表，便于复核和展示。
4. 通过可视化大屏集中呈现核心指标和主要结论。
5. 根据分析结果形成商业解读和客户挽留建议。

## 三、数据概况

- 客户记录：10,000 条
- 字段数量：15 个
- 流失客户：2,037 人
- 整体流失率：20.37%
- 目标字段：`Exited`，其中 `1` 表示客户已流失，`0` 表示客户未流失
- 主要字段：信用评分、地区、性别、年龄、在行年限、账户余额、产品持有数量、是否持有信用卡、是否为活跃客户、预估薪资等
- `EB` 字段：客户流失预测结果字段，已包含在 `银行客户流失预测数据.csv` 中

## 四、项目分析流程

1. **数据准备**：使用 `银行客户流失预测数据.csv` 作为客户明细和预测结果数据。
2. **SQL 分析**：运行 `银行客户流失查询.sql`，计算全局指标和不同客群的流失率。
3. **结果整理**：将关键查询结果分别保存为全局、地区性别、资产、产品和活跃度分析表。
4. **可视化展示**：使用查询结果搭建银行客户流失可视化大屏。
5. **商业分析**：结合指标差异识别重点客群，并在商业分析报告中提出运营建议。

## 七、SQL 分析内容

`银行客户流失查询.sql` 基于 `bank_clean` 数据表完成以下分析：

1. **全局流失情况**：计算全部客户数、流失客户数和整体流失率。
2. **地区与性别分析**：通过 `Geography` 和 `Gender` 交叉分组，对比不同人群的客户数量与流失率。
3. **年龄分层分析**：使用 `CASE WHEN` 将客户划分为 30 岁以下、31-40 岁、41-50 岁、51-60 岁和 60 岁以上五个年龄段。
4. **资产分层分析**：根据 `Balance` 将客户划分为 0 余额、0-5 万、5-10 万和 10 万以上四个资产层级。
5. **产品数量分析**：按 `NumOfProducts` 统计不同产品持有数量对应的流失率。
6. **客户活跃度分析**：对比活跃客户与沉睡客户的流失差异。

## 八、核心分析发现

- 数据集中共有 2,037 名流失客户，整体流失率为 20.37%。
- 德国客户的流失率约为 32.44%，高于法国和西班牙客户，是地区维度需要重点关注的人群。
- 年龄、账户余额、产品持有数量和客户活跃状态均可用于进一步识别高流失风险客群。
- 客户运营应结合多个维度进行分层，而不是只依赖单一特征判断客户风险。

更完整的指标解读、风险客群总结和运营建议见 `商业分析报告/`。

## 十、项目使用方法

### 查看分析结果

直接打开 `银行客户流失分析表/` 中对应的 XLS 文件，即可查看不同维度的统计结果；客户明细和预测结果位于 `银行客户流失预测数据.csv`。

### 运行 SQL 查询

1. 在 MySQL 中准备名为 `bank_clean` 的客户数据表。
2. 确保表中包含 `Exited`、`Geography`、`Gender`、`Age`、`Balance`、`NumOfProducts` 和 `IsActiveMember` 等字段。
3. 执行 `sql语句查询/银行客户流失查询.sql`。
4. 查看各查询对应的流失率统计结果。

### 查看可视化大屏

1. 在 GitHub README 中查看静态大屏截图。
2. 点击“在线查看银行客户流失可视化大屏”打开在线版本。
3. 或下载项目后，直接使用浏览器打开 `数据化大屏/银行客户流失可视化大屏.html`。

### 查看商业分析

打开 `商业分析报告/` 中的商业分析报告，查看核心发现、客群特征、业务影响和客户挽留建议。
